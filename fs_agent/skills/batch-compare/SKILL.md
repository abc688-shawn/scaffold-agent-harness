# 批量对比技能

## 何时使用

用户希望比较两个或多个文件，找出内容上的异同：
- "对比一下 v1.txt 和 v2.txt"
- "这两份报告有什么区别"
- "找出这几个配置文件的不同"
- "哪里变了"

## 不要误用的场景

- 用户只想了解单个文件内容（用 doc-qa 技能）
- 用户想整理文件（用 file-organize 技能）
- 对比的是目录结构而非文件内容（直接用 list_files）

## 核心框架

对比任务分为三步：**读取 → 分析 → 呈现**。

### 第一步：读取（Read）
1. 用 `file_info` 确认两个（或多个）目标文件存在
2. 依次 `read_file` 读取各文件内容
3. 若文件较大，用 `search_in_file` 按章节/关键词定向读取

### 第二步：分析（Analyze）
针对文件类型选择对比维度：
- **文本/Markdown/代码**：逐段比较，找新增、删除、修改的段落
- **CSV/JSON**：比较字段/键的增删、值的变化
- **配置文件**：重点关注参数值的差异
- **报告类文档**：比较结论、数据、章节结构

### 第三步：呈现（Present）
用清晰的结构输出对比结果：
- 先给出**总体评估**（差异程度：微小 / 局部 / 大幅重写）
- 再列出**具体差异**（按重要程度降序）
- 如有共同点，最后说明**一致的部分**

## 输出结构建议

```
**总体评估**：两个文件在 X 方面有 Y 处差异，整体变动 [小/中/大]。

**主要差异**：
| 项目 | 文件A | 文件B |
|------|-------|-------|
| …    | …     | …     |

**新增内容**（仅 B 有）：…
**删除内容**（仅 A 有）：…
**共同内容**：…
```

## 示例问题

- "对比 report_v1.pdf 和 report_v2.pdf，有哪些变化"
- "config_dev.json 和 config_prod.json 有什么不同"
- "找出这两个 Python 文件的区别"
