# 文档问答技能

## 何时使用

用户希望了解某个或某类文件的内容，例如：
- "这份 PDF 讲了什么"
- "帮我总结一下这几个文档"
- "这个文件的主要内容是什么"
- "归纳一下 report.txt 的要点"

## 不要误用的场景

- 用户想整理文件结构（用 file-organize 技能）
- 用户想对比两个文件的差异（用 batch-compare 技能）
- 用户只想知道文件是否存在（直接用 list_files / file_info）

## 核心框架

文档问答分为三步：**定位 → 阅读 → 回答**。

### 第一步：定位（Locate）
1. 如果用户指定了文件名，先用 `file_info` 确认文件存在并获取大小
2. 如果用户未指定，用 `list_files` 找候选文件，结合上下文判断最可能的目标
3. 对于大文件（>50KB），优先用 `search_in_file` 按关键词定位章节再读取

### 第二步：阅读（Read）
1. `read_file` 读取文件内容
2. 若文件为 PDF/DOCX（二进制），内容会由工具自动提取为文本
3. 若文件过长被截断，说明截断情况并基于已读内容回答

### 第三步：回答（Answer）
- 回答要结构化：先给出**一句话摘要**，再展开**要点列表**
- 引用文件中的原文片段作为依据（加引号标注）
- 如信息不足，明确说"基于目前读取的内容…"并建议用户进一步指定范围

## 输出结构建议

```
**一句话摘要**：本文档介绍了…

**主要内容**：
1. …
2. …
3. …

**关键数据 / 结论**：
- …
```

## 示例问题

- "report.pdf 讲了什么，帮我总结一下"
- "这个目录下有几个文档，分别讲什么"
- "meeting_notes.txt 的主要结论是什么"
